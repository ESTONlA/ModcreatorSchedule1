# Contributing
Welcome potential contributor! 
Thank you for your interest in helping improve the Schedule1 Modding Tool.
Please read over the below in full to help you get started

## Important
- Please thoroughly read over [CODING_STANDARDS.md](CODING_STANDARDS.md) before contributing.

## PR Preparations
Verify your changes will successfully build and run prior to PR please.
Ultimately, this just saves you time and gets your changes merged faster.

Before submitting a PR, ensure:
- ✅ Project builds without errors
- ✅ Application launches without crashes
- ✅ New features work as expected
- ✅ Existing features still work
- ✅ Generated code is valid C# and compiles
- ✅ UI changes follow Material Design patterns
- ✅ Save/Load preserves data correctly

## Proper Contributing Channels
All pull requests should include:
- **Description**: What does this PR do?
- **Motivation**: Why is this change needed?
- **Changes**: List of key changes made
- **Testing**: Did you test this?
- **Screenshots**: Optionally, if UI changes, include before/after screenshots
- **Related Issues**: Link to any related GitHub issues

**PR Title Format:**
- `[Feature] Add Location blueprint support`
- `[Fix] Resolve schedule time validation bug`
- `[Refactor] Simplify NpcCodeGenerator structure`
- `[UI] Improve NPC editor tab layout`

## Code Quality Expectations
- Follow MVVM pattern consistently
- Write clean, self-documenting code
- Add XML documentation for public APIs
- Maintain Material Design UI consistency
- Validate user input and handle errors gracefully
- Generate valid, well-formatted C# code from blueprints
